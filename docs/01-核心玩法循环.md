# xiaochu2 核心玩法循环设计

> 原型：xiao_chu（灵宠消消塔，已上线微信小游戏）
> 目标：解决原版「玩法冗余、即时反馈弱、目标感不强」三个问题
> 平台：微信 / 抖音小游戏

## 1. 核心循环

```
关卡战斗（转珠 + Combo） → 必得灵宠币 → 招募灵宠（重复自动转碎片升星）
        ↑                                          ↓
   产出更多灵宠币  ←  挑战更强 / 克制关卡  ←  战力提升 + 克制属性覆盖
```

与对标游戏「智龙迷城」同构（地下城 → 掉宠物/魔法石 → 喂养进化 → 更深地下城），
但针对小游戏平台做两点关键修正：

### 修正 1：单货币双用途的目标冲突

原循环里灵宠币既「招募新宠」又「升级旧宠」，玩家每次拿到币都要纠结，目标模糊。

**修法：灵宠币只做一件事 —— 招募。**

- 招募是唯一主动消费决策，进度条常驻主页（「下一只灵宠：还差 N 币」）
- 重复招募 → 自动转化为该宠碎片 → 碎片够了自动提示升星（副产物，零决策成本）
- 宠物升级消耗「经验」，由关卡战斗直接掉落，低决策投入

货币收敛（原版 8 种 → 3 种）：

| 资源 | 用途 | 获取 | 是否展示为货币 |
|---|---|---|---|
| 灵宠币 | 招募灵宠 | 每关必得（通关 / 三星目标加成） | 是（主货币） |
| 体力 | 进关门槛 | 自然恢复 / 看广告 | 是 |
| 碎片 | 升星 | 重复招募自动转化 | 否（仅在宠物详情展示进度） |

### 修正 2：纯数值墙易流失

「更强关卡」如果只是 HP 膨胀，卡关 = 流失。

**修法：五行克制做软墙。**

- 关卡有主属性，克制属性伤害 ×1.6、被克 ×0.5
- 卡关时明确提示：「这关是水属性，招募 / 上场土系灵宠」
- 「换队伍」与「养数值」都是过墙手段 → 卡关反而强化招募目标感

## 2. 战斗规则（继承原版已验证部分）

- 6×6 棋盘，5 属性珠（金木水火土）+ 心珠（回血）
- 长拖转珠：按住一颗珠沿任意路径拖动，沿途交换，限时 12 秒
- 消除 3+ 同色珠 → 对应属性灵宠攻击；一次拖珠消除多组 = Combo
- Combo 伤害倍率：每多 1 连 +20%（递减分段，见数值框架文档）
- 4 连消 ×1.5、5 连消 ×2.0
- 敌人回合制反击，心珠回血是生存手段

## 3. 即时反馈设计

| 时机 | 反馈 |
|---|---|
| 消除瞬间 | 珠子爆裂粒子 + 音效 |
| 每组消除 | Combo 计数实时跳动 + 伤害倍率显示（「3 Combo ×1.4」） |
| 结算攻击 | 宠物 Sprite 冲刺位移 → 敌人受击闪白 + 抖动 → 属性色斩击特效 → 伤害飘字 |
| 高 Combo | 全队依次连续冲刺，「群殴」演出 |
| 通关 | 灵宠币逐枚飞入钱包动画 + 三星结算 |

## 4. 目标感设计

- **主页常驻**：「下一只灵宠」招募进度条（明示商店直购 / 轮换，不做黑箱抽卡）
- **每关三星目标**：通关 / 二星回合上限 / 三星回合上限（更快），星数追加灵宠币
- **章节 Boss**：掉落限定灵宠（唯一非商店获取渠道，制造期待）
- **新手前 3 关**：教会三件事 —— 转珠=攻击、连击=高伤、宠物搭配（继承 xiao_chu 改版方案）

## 5. 首版范围（MVP）

**只做**：主线关卡 + 灵宠招募/升星/升级一条养成线 + 灵宠币。

**砍掉**（原版冗余系统，后续按数据决定是否加回）：

- 通天塔 Roguelike、天机试炼（三套 PVE 壳合并为一套关卡框架）
- 修炼（五维加点）、法宝（第二三条养成线）
- 派遣挂机、图鉴里程碑、双轨签到
- 局内宠物合并升星（与局外升星二选一，保留局外）
