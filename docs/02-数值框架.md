# xiaochu2 数值框架设计

> 核心原则：数值（balance）、公式（formulas）、逻辑（game）三层解耦，单一真源，可单测。

## 1. 三层架构

```
src/balance/   纯数据（TS 常量表，零逻辑、零 import 业务代码）
src/formulas/  纯函数（零状态、零副作用，输入输出全显式，可单测）
src/game/      逻辑层（只调 formulas + 读 balance，禁止 magic number）
```

依赖方向严格单向：`game → formulas → balance`，反向禁止。

### 对照 xiao_chu 的问题修正

| xiao_chu 问题 | xiaochu2 做法 |
|---|---|
| 宠物数值 `pets.js` 字面量 + `petBase.js` 启动时 merge 双写 | 宠物表只在 `balance/pets.ts` 一处，含数值与文案 |
| `balance → economyConfig → constants` 三层转发 | balance 文件直接被引用，无中转层 |
| ★3/★5 技能 desc 与数值分两个文件 | 同一条记录内聚 |
| `stages.js` 1000+ 行含硬编码奖励逻辑 | 关卡表只存参数，奖励由公式生成 |
| 战斗布局/动画帧数散落 UI 层 | UI 常量独立 `balance/ui.ts`，不与数值混放 |

## 2. balance 文件清单

| 文件 | 内容 |
|---|---|
| `combat.ts` | Combo 倍率分段、消除数倍率、克制倍率、暴击、英雄基础 HP、拖珠时限 |
| `pets.ts` | 灵宠表：id / 属性 / 稀有度 / 基础攻击 / 成长系数 / 技能（数值+文案内聚） |
| `enemies.ts` | 敌人模板：基值 + 技能引用 |
| `stages.ts` | 关卡表：章节 / 主属性 / 敌人引用 / 难度系数（数值由曲线生成） |
| `economy.ts` | 灵宠币产出曲线参数、招募定价、升星碎片消耗、体力 |
| `growth.ts` | 等级经验曲线、星级倍率参数 |
| `ui.ts` | 棋盘布局、动画帧数等表现常量（与数值平衡隔离） |

## 3. 关键公式（formulas/）

### 伤害管线（damage.ts）

```
基础攻击 = Σ(同属性宠物攻击) × 消除数倍率(3/4/5连)
最终伤害 = 基础攻击 × Combo倍率 × 克制(×1.6 / ×0.5) - 敌防 → 暴击
```

Combo 倍率（递减分段，继承原版已验证手感）：

| Combo 区间 | 每段加成 |
|---|---|
| 2 ~ 6 | +20% |
| 7 ~ 10 | +15% |
| 11+ | +8% |

示例：3 Combo → ×1.4，6 Combo → ×2.0，10 Combo → ×2.6。

### 成长曲线（growth.ts）

- 宠物攻击 = 基础攻击 × (1 + 等级成长率)^(Lv-1) × 星级倍率
- 敌人数值 = 章节基值 × 章节成长系数^(章节-1) × 关内递增系数
- 调难度只改曲线参数，不改表

### 经济产出（economyOutput.ts）

- 单关灵宠币 = 基础产出 × 章节系数 × 星数加成
- 招募定价随已招募数量递增（前期快、后期稳），保证「下一只」目标始终可见
- 约束：玩家正常推图速度下，每 15~25 分钟完成一次招募（首日 4~6 只）

## 4. 单测策略（vitest）

- `formulas/__tests__/damage.test.ts`：Combo 倍率断点快照、克制计算、防御分摊
- `formulas/__tests__/growth.test.ts`：成长曲线快照（改参数 → 测试 diff 即全局影响一览）
- `formulas/__tests__/economy.test.ts`：产出/消耗平衡断言（如「第一章总产出 ≥ 前 3 只招募定价」）

改数值的工作流：改 `balance/` → `npm test` → 看快照 diff 确认影响范围 → 提交。
